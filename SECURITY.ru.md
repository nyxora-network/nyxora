## Сообщение об уязвимости

Мы серьёзно относимся к безопасности. Если вы обнаружите уязвимость в NYXORA,
**не откройте** публичный issue.

Вместо этого отправьте отчёт в наш Telegram канал: [@NyxoraCore](https://t.me/NyxoraCore)

Мы подтвердим получение в течение 48 часов и предоставим подробный ответ в течение 7 дней.

### Что указать

- Описание уязвимости
- Шаги для воспроизведения
- Затронутые версии
- Возможное влияние
- Предлагаемое исправление (если есть)

## Лучшие практики безопасности

При использовании NYXORA:

1. **Используйте сильные пароли SSH** или аутентификацию по ключу
2. **Измените пароли по умолчанию** — NYXORA автоматически генерирует пароли для Shadowsocks, Rathole, Hysteria, Backhaul и IPsec, но вы должны установить их явно через переменные окружения для producción
3. **Правила файрвола** — `nyxora install` и процесс подключения настраивают правила iptables для ограничения доступа к туннелям
4. **Регулярно обновляйтесь** — запускайте `nyxora update` или проверяйте страницу релизов GitHub
5. **Запускайте от непривилегированного пользователя** когда это возможно (хотя настройка туннеля требует root для WireGuard и iptables)
